📊 Cadeia de Valor e Mapa Estratégico CAGE

CAGE-RS — Indicadores e Processos · Power BI
🔗 Cadeia de Valor
A cadeia de valor representa, de forma estruturada, as principais atividades que a instituição realiza para gerar valor à sociedade. Ela serve para entender como os processos se conectam, navegar na arquitetura de processos, alinhar as ações ao planejamento estratégico e identificar oportunidades de melhoria e inovação na entrega dos serviços públicos, figurando como uma valiosa ferramenta de apoio à tomada de decisão.
🧭 Mapa Estratégico
O Mapa Estratégico é uma representação gráfica que organiza e comunica os objetivos estratégicos de uma organização de forma clara e estruturada. Ele mostra como diferentes metas estão conectadas e contribuem para o alcance da visão e missão da empresa, facilitando o alinhamento entre áreas e a tomada de decisões. Essa ferramenta ajuda a transformar a estratégia em ações concretas, promovendo foco e coerência na execução.
O painel requer autenticação com sua conta Microsoft. Caso não carregue, abra em nova aba.

Leia o documento INTEIRO antes de estruturar os dados. Nunca invente informações. Se um campo não existir no documento, retorne null ou lista vazia. Retorne APENAS JSON puro, sem explicações. # REGRA CRÍTICA — LIMPEZA 1. Todos os campos começam vazios. 2. Nunca reutilize informações de processos anteriores. 3. Cada documento gera um JSON totalmente independente. # SIGLAS Extraia TODAS as siglas e abreviações presentes no documento (ex: CAGE, CGE, TCE, POP, SAEWEB, SEI, SIAPC). Para cada uma inclua o significado completo como "descricao". Não invente siglas ausentes no documento. # EXECUTORES DO PROCESSO REGRA CRÍTICA: crie uma entrada SEPARADA em executores[] para CADA executor. Se o documento listar "Auxiliar administrativo; Analista; Auditor", gere 3 objetos distintos — NUNCA em uma única entrada. No campo "actor" de CADA action dentro de steps[], use EXATAMENTE o mesmo texto do campo "name" do executor responsável por aquela ação. O campo "title" deve conter o cargo/função do executor. O campo "duties" deve conter as responsabilidades gerais desse executor (não as atividades do processo). # CLASSIFICAÇÃO DAS AÇÕES (campo "type") - atividade: ação executável (ex: "Registrar processo", "Encaminhar relatório") - decisao: ponto de escolha binário/múltiplo (ex: "Documentação completa?", "Processo novo?") - evento: marco de início ou fim (ex: "Receber solicitação", "Processo encerrado") - observacao: explicações, comentários, OBS:, textos entre parênteses NUNCA classifique perguntas ou comentários como "atividade". # FLUXO DETALHADO (campo flow[]) Mapeie TODOS os passos do processo SEM simplificar.
